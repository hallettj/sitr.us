<Title text="Open Source Projects" />

[git-format-staged](https://github.com/hallettj/git-format-staged/blob/master/README.md) —
A git add-on that formats staged changes to code files
using an automatic formatter like [Prettier](https://prettier.io/),
and works seamlessly with partially-staged files.
There are other tools that do the same job;
but git-format-staged takes a unique, and in my opinion elegant, approach.
See [my blog post](https://www.olioapps.com/blog/automatic-code-formatting/) for details,
and for some background on git's internal workings.

[redux-instant](https://github.com/hallettj/redux-instant/blob/master/README.md) —
Minimize boilerplate in your [Redux](https://redux.js.org/) app
by inferring action creators and action types from your reducer
with precise type-checking when you use Typescript.
Actions created by this library are serializable,
and are compatible with time-travel debugging.
